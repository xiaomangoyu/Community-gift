# 应援棒工作流理解

这份文档记录当前阶段对“社群礼物 / 主播应援棒”工作流的共同理解，作为后续接入真实主播数据前的设计基准。

## 核心目标

最终产物是给主播/社群定制的单支应援棒图片。

图片目标不是普通 logo、海报、贴纸或头像，而是：

- 黑色背景上的单支完整应援棒产品图
- 有收藏级 3D 产品渲染质感
- 有清楚的灯头、连接件、手柄、底部节点
- 主播/社群文字自然嵌入灯头核心
- 颜色、符号、材质、形态能反映主播气质
- 效果接近 reference image 中 Strong 行的高级收藏玩具感

## 重要共识：应援棒不是只有爱心

“应援棒”是统一品类，不是统一爱心模板。

爱心是其中一种常见母题，适合表达浪漫、甜美、粉丝爱意，但不能让所有主播都变成同一种爱心棒。

后续逻辑应该先判断主播适合的灯头主体形态，再决定颜色、符号、材质、文字和装饰。

## 重要共识：基础稳定后，创意轴是关键变量

当前方向不是让最终 prompt 重新变成自由发挥，而是在稳定基础模板之上增加一层可解释、可开关的创意变量。

可以把整体分成两层：

```text
基础层 baseline
  黑底、完整单支应援棒、灯头核心、连接件、手柄、底盖、唯一 exact text

创意层 creative_axes
  根据主播数据决定是否更发散，以及往哪个方向发散
```

“野性”只是创意层中的一个轴，不是唯一目标。它适合黑豹、鹰、乌鸦、龙、牛角、拳击等主播，会影响外轮廓、羽翼、角爪、鳞片和前探张力。但其他主播可能更适合可爱、奢华、音乐、战队、浪漫、清爽、神秘、街头或治愈。

这层变量的原则：

- 默认不开启，先保住基础产品结构和文字安全。
- 只有数据、VLM facts 或人工/研发字段支持时才打开。
- 创意判断先以气质为主，轮廓和姿态辅助落地。
- 材质暂时不进入 creative axes，只跟随 reference / template 的材质槽位。
- VLM 只负责提取事实，不负责决定创意强度。
- mapper / routers 把事实映射成结构化 `creative_profile`。
- template-first prompt 只消费结构化 profile，不直接吃创意散文。

未来研发可以在 `signals.md` 里追加可选控制块：

```md
## Creative controls
- **creative_mode**: baseline | expressive | wild
- **wildness_score**: 0-3
- **wildness_axes**: horn, claw, spike, swept_wing, rock_glam
```

更通用的长期接口可以是：

```json
{
  "creative_mode": "baseline | expressive | bold",
  "creative_intensity": 0,
  "creative_axes": ["wild", "luxury", "music"],
  "axis_details": ["swept_wing", "lightning", "upward"]
}
```

参考创意轴：

| 创意轴 | 典型场景 | 影响 |
| --- | --- | --- |
| `wild` | 黑豹、鹰、乌鸦、龙、牛角 | 更外扩、更锋利、羽翼/角/爪/鳞片更强 |
| `cute` | 蜜瓜、土豆、龟、猫、云朵 | 更圆润、果冻、奶油色、软萌比例 |
| `luxury` | 皇冠、女王、阿拉伯金绿、珠宝 | 宝石核心、细边轮廓、徽章感、收藏级层次 |
| `music` | DJ、麦克风、rapper、唱歌 | 声波、麦克风轮廓、夜场光、舞台动势 |
| `battle` | fighter、police、PK、Free Fire | 徽章、护片、能量环、战队秩序感 |
| `romantic` | 爱心、蝴蝶、粉紫、陪伴 | 柔光、透明宝石、丝带和心形边缘 |
| `fresh` | 蜜瓜、海岛、薄荷、青柠 | 半透明果冻、浅绿、玻璃、叶片 |
| `mystic` | 阿拉伯、沙漠、古金、龙 | 古金、沙岩、宝石、卷曲结构 |
| `street` | 棒球帽、rapper、neon、游戏 | 前压帽檐、霓虹涂层、运动感 |
| `cozy` | 房间、聊天、陪伴、小动物 | 短绒、奶白、柔雾、圆胖结构 |

## 可用主体形态

### 心形

适合：

- 甜美
- 浪漫
- 亲切
- 粉丝爱意强
- 情绪表达直接

视觉语言：

- 圆润爱心核心
- 层叠爱心外轮廓
- 小心形节点
- 珠光、果冻、糖果树脂

### 圆章 / 徽章形

适合：

- 通用型主播
- 稳定社群
- 直播间 logo 感
- 需要清楚承载名字

视觉语言：

- 圆形或圆角徽章灯头
- 中央发光铭牌
- 外圈珠光或金属包边
- 结构稳定、识别清楚

注意：不能做成平面徽章，必须保留灯头厚度和手柄结构。

### 皇冠形

适合：

- 女王感
- 顶流感
- 强势
- 华丽
- 舞台中心型主播

视觉语言：

- 冠冕尖角
- 对称高光结构
- 珠光/钻石/金属点缀
- 粉、玫红、金、银等高识别舞台色

### 翅膀形

适合：

- 梦幻
- 守护感
- 天使感
- 温柔但有记忆点
- 偶像/女团气质

视觉语言：

- 两侧展开的翅膀大轮廓
- 中央发光核心
- 羽片作为实体树脂或珠光片
- 轻盈但不能薄成纸片

### 花朵 / 水晶形

适合：

- 温柔
- 精致
- 优雅
- 礼物感
- 浪漫但不一定要爱心

视觉语言：

- 花瓣包裹核心
- 水晶或宝石切面
- 珠光、半透明树脂
- 柔和边缘光

### 水果 / 糖果形

适合：

- 可爱
- 活力
- 年轻
- 甜酷
- 高亲和力

视觉语言：

- 果实或糖果作为灯头轮廓
- 饱满圆润体块
- 高饱和颜色
- 果冻、糖果漆、半透明晶体

### 火焰 / 流线形

适合：

- 高能
- 舞台
- 热血
- 竞技
- 直播冲击力

视觉语言：

- 上扬火焰或速度流线
- 明确前冲轮廓
- 红、橙、金、电光色
- 高光树脂和能量晶体

注意：要像应援棒，不要变成武器或电竞道具。

### 海浪 / 贝壳形

适合：

- 海岛
- 旅行
- 清爽
- 度假感
- 热情轻松

视觉语言：

- 波浪包裹灯头
- 贝壳、珍珠、海风曲线
- 蓝绿、珊瑚、珠光白
- 清透晶体、果冻树脂

### 机械 / 装甲形

适合：

- 电竞
- 酷感
- 力量感
- 秩序感
- 对抗感

视觉语言：

- 少量机械铰链
- 对称锁扣
- 金属护片
- 黑红、银灰、电光色

注意：机械元素必须克制。不能出现武器感、工业工具感、过多碎片、过多尖刺或冷硬金属占主导。

## Strong Reference 的审美公式

当前更认可的目标不是“固定 45 度心形框”，而是新 reference 合集中 Strong 行呈现的整体质量：

- 一眼能看出是高级应援棒
- 灯头主体轮廓强，缩略图也能识别
- 主题装饰长在产品外轮廓上，而不是贴在表面
- 中央核心是饱满发光实体，不是空心或镂空
- 文字嵌入或浮雕在中央核心里
- 手柄和底部节点完整
- 材质偏圆润收藏玩具：树脂、珐琅、果冻晶体、珠光、克制金属
- 黑底干净，局部发光克制

## Weak Reference 需要避免

从 Weak 行总结出的反例：

- 过度机械化
- 武器感或工业工具感
- 太多碎片、尖刺、支架、小零件
- 中央核心空、薄、像 logo 框
- 手柄像随便接上的普通杆子
- 文字贴在表面，不像产品结构的一部分
- 整体像徽章、镜子、海报或 UI 图标，而不是应援棒
- 底部节点没设计或被裁切
- 材质太平、太硬、太灰、太金属

## 主播数据到图片的逻辑

后续接真实主播信息时，推荐线性流程：

1. 读取主播原始信息
2. 翻译/清洗多语言内容
3. 只提炼名字、颜色、元素和氛围
4. 编排层判断主体形态
5. 编排层选择材质、复杂度和禁用项
6. 把符号转译成产品结构
8. 生成 Seedance/Seedream prompt
9. 出图
10. 人工复看 5 张小样，记录共性问题
11. 按共性问题改 prompt 模板或工作流

VLM 硬约束检查暂时不进入当前主流程。等人工 pass / not pass 标准稳定后，再把 VLM 作为批量质检和失败类型分类工具接回。

## 主播信息映射重点

真实主播信息进入后，最重要的不是把所有信息塞进图里，而是做取舍。

每个主播应主要映射到：

- 主播名字
- 主色
- 辅助色
- 1-2 个符号
- 直播氛围

主体形态、材质、装饰强度、禁用项和输出类型都交给编排层兜底，不在简化 CSV 里提前写死。

当前已验证的 baseline 方向：

- 黑底
- 单支完整应援棒
- 高光树脂 / 果冻晶体 / 珠光

- 装饰围绕灯头轮廓生长
- 文字嵌入核心

- 底部黑边更舒服
- 手柄和底部节点更定制
- 不同主播的主体形态更有差异
- 外轮廓符号更大胆，但不能杂乱
